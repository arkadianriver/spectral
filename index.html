COUPON ADMIN SYSTEM

쿠폰 어드민의 Front-end 구성은 BootStrap과 Jquery를 기반으로 운영 편리성을 고려하고
Angular Js, Ajax 등을 통해 SpringFrameWork 기반의 Back-end 서버와 통신하였습니다.
그리고 쿠폰 생성/수정 시, 사용용도에 따라 저장소를 다르게 사용했습니다.
저장소에 대한 설명은 아래 모식도 및 하단에 상세 설명이 있습니다.
또한 Meta Channel에 노출되는 쿠폰 정보를 관리 할 수 있도록 했습니다.

mysql은 내부 관리 데이터 저장소로, 가장 최신의 데이터를 반영하고 있습니다.
따라서 모든 원천 데이터는 mysql rdb 정보를 기준으로 정리되도록 합니다.
쿠폰에 대한 정책정보 외 수량, 발급, 대상, 로그 등의 데이터를 저장하도록 했습니다.
java에서 받은 data params을 mybatis를 통해 sql query 를 mapping 시켰습니다.
각 mapper file들은 package 단위로 관리하며, 각각의 repository에 연결시켰습니다.


쿠폰정보를 캐싱하여 사용자 클라이언트에서 사용하기 위한 용도입니다.
클라이언트에서 요청이 오면, API에서 해당 정보를 통해 결과값을 내려줍니다.
쿠폰의 생성/수정 시, rdb의 원천정보와 일치하는지 확인 후 결과값을 저장시켰습니다.
레디스를 사용하기 위해, redis clients인 jedisCluster를 사용하여 시스템을 구성했습니다.
쿠폰의 정책정보 외 적용대상에 대한 각각의 key/value 구조를를 구성함으로써
API에서 해당 정보를 통해 빠르고 정확한 데이터를 가져올 수 있도록 했습니다.


쿠폰의 부가적인 정보는 couchbase를 통해 저장 및 갱신했습니다.
쿠폰의 혜택, 수량, 계산된 할인가 정보를 Json Document 기반으로 저장했습니다.
이 또한 캐싱 데이터로써, 위 정보들은 클라이언트 user view에 결과값을 내려줍니다.
특히, 쿠폰 어드민에서는 쿠폰의 승인여부에 따라 직접 couchbase document를 관리했습니다.
그리고 쿠폰의 대상이 변경된 경우, couchbase 정보를 갱신하기 위해 redis에 갱신을 위한
쿠폰번호를 저장함으로써, 갱신용 Batch 작업을 통해 couchbase 정보를 갱신시켰습니다.


쿠폰의 갱신시, meta channel(제휴사)에 노출되는 상품의 쿠폰에 대해 동기화가 필요합니다.
이를 해결하기 위해 RabbitMq를 이용하였으며, meta channel의 상품 중 쿠폰이 적용된
딜(상품)에 대한 변경이 있을 시 동기화를 위해 딜번호를 message queue에 전송시켰습니다.
해당 message queue 정보를 기반으로 meta channel에 대한 쿠폰정보를 갱신하는 Batch와 함께
주기적으로 제휴사 쿠폰 정보를 갱신시킬 수 있게 구성했습니다.

Project 02.

TARGETING COUPON SYSTEM

쿠폰은 사용자가 필요에 의해 버튼을 통한 click event로 발급받는게 일반적입니다.
하지만, 쿠폰함에 자신이 받지 않았던 쿠폰이 갑자기 있다면 어떨까요?
사실상, 사용자는 자신에게 손해가 오는 입장이 아니라면 이를 버그라고 생각하지 않을겁니다.
본인이 직접 발급받지 않았지만, 확인 후 사용가치가 있는 쿠폰이라면 충분히 사용할 수 있을겁니다.
즉, 다양한 스킴의 쿠폰혜택을 사용자의 구매 패턴에 따라 구분하고 개개인마다 타겟을 설정하여
각자의 쿠폰함에 쿠폰을 직접 발급받지 않고 시스템을 통해 꽂아주도록 설계된 시스템입니다.

운영 및 유지보수 기간 : 6개월

TARGETING COUPON SYSTEM

쿠폰의 혜택 스킴에 따라 고객군을 나누어, 각 고객에 대한 타겟팅 된 쿠폰정보를 설정합니다.
이는 hadoop driver를 통해 연결되어 있는 jdbc hive에 대량의 데이터로 전달받습니다.
이때 정보는 대상자에 대한 key와 매칭되는 쿠폰에 대한 key가 각각 row 단위로 구성되어 있습니다.
hive에 들어있는 데이터는 hiveQL을 통해 SQL 연산으로 Spring Batch를 통해 시킵니다.
spring batch의 read/write 작업을 통해 대량의 데이터를 partitioning 하여 중간테이블을 만들어
원본데이터의 그룹 파티셔닝이 된 테이블을 만듭니다. 이후, 파티션별로 발급을 진행하며
실제로 발급된 쿠폰은 mysql의 쿠폰발급, 로그 테이블에 각각 데이터를 insert합니다.


전 고객대상으로 쿠폰발급을 진행 할 경우, 많게는 한번에 3천만장의 쿠폰을 발급했습니다.
이는, 실제로 batch system을 통해 발급할 시, 물리적 시간으로만 최소 3~4일이 소요됬습니다.
쿠폰의 중복, 오발급, 누락등의 이슈가 발생될 경우 정산에도 문제가 발생할 수 있기때문에
쿠폰 자체에 사용자에게 노출이 되는 시점을 두고, 천천히 발급 할 수 있도록 시스템을 설계하여
안정적으로 대량의 데이터를 read/write 할 수 있도록 하는것이 중요한 요소 중 하나였습니다.

Project 03.

POMPOM SNS SYSTEM

졸업한 K대학의 교내 SNS를 만들었고, 사이트 이름은 POMPOM으로 지었습니다.
교내에는 자신의 학과 외 타과 및 동아리, 교내소식 등을 교류할 수 있는 공간이 없었습니다.
그래서 교내 SNS를 만들고자 했으며, PC와 모바일웹 지원 및 페이스북 인증 가입을 통해
간편하고 빠른 접근성과 보안성을 동시에 높일 수 있었습니다.
오픈과 동시에 교내 600명 이상의 회원을 확보하며, 다양한 교류를 할 수 있도록 제공 했습니다.

개발기간 : 3 ~ 4개월

POMPOM

User Interface를 고려하여 PC와 모바일 웹으로 서비스를 구성하였습니다.
사용자들은 개개인의 타임라인, 친구들의 게시글이 스크롤방식으로 제공되는 뉴스피드
자신의 팔로잉을 관리 할 수 있는 친구 관리 및 관심사나 개인정보에 따른 친구추천 화면
그리고 메인 홈 페이지에서는 학과의 기숙사, 학생식당, 버스정보 등 다양한 교내 정보를
단순히 보는것뿐 아니라 유저들의 피드백을 통해 관리 될 수 있도록 구성했습니다.

POMPOM SNS는 기본적으로 사용자 수가 교내 수준이며, 빠르게 서버를 구현기위해
node js를 server side language로 선택했습니다. 또한 다양하게 제공되는 npm 모듈을 통해
여러가지 기능을 손쉽게 제공 받을 수 있었으며, 특히 express module을 중심으로 처리했습니다.
가입절차는 페이스북 연동과 교내 학번 연동을 통해, 간편한 프로세스로 가입이 가능했습니다.
이를 통해, 접근성과 편리성을 고려함과 동시에 개인정보에 대한 이슈도 해결할 수 있었습니다.
화면구성은 bootstrap을 기본으로하여, 필요한 css, js component를 import하여 사용했습니다.
교내 사이트에서 제공되는 식단, 버스정보 등을 crawling 하고 데이터를 가공함으로써
제공된 정보로 사용자들끼리 다양한 의견을 주고 받을 수 있는 공간을 만들어줬습니다.
