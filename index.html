Coupon Admin System

쿠폰 어드민은 운영의 편리성을 고려한 단순하고 명확하게 화면을 설계했으며
쿠폰에 적용되는 복잡하고 다양한 정책정보를 쉽게 컨트롤할 수 있게 구성했습니다.
쿠폰 생성/수정 시, 사용 용도에 따라 저장소를 다르게 사용했습니다.
각 저장소의 고유 특징을 통해, 쿠폰의 다양한 정책정보를 용도에 맞게 저장합니다.

  • mysql
  • - mysql은 내부 관리 데이터 저장소로, 가장 최신의 원천 데이터를 관리
  • - spring server에서 mybatis mapper를 통한 sql 관리

  • redis
  • - redis clients jedisCluster를 사용하여 레디스 연결
  • - 쿠폰의 수량, 적용대상, 만료 번호 등을 저장하여 각종 api, batch를 위한 데이터 저장
  • - rdb와 memory에 쿠폰의 정책정보를 이원화 저장하여 관리데이터와 사용데이터를 나눔

  • couchbase
  • - 쿠폰의 혜택, 계산된 가격 정보 등을 Json Document 기반으로 저장 및 관리
  • - user client side에서 사용 될 데이터 정보들을 저장
  • - batch 시스템을 통해 갱신될 데이터들은 redis정보를 기반으로 변경

  • rabbit mq
  • - 쿠폰의 갱신시, meta channel(제휴사)에 노출되는 상품의 쿠폰에 대해 동기화가 필요
  • - 딜(상품)에 대한 변경이 있을 시 동기화를 위해 딜번호를 message queue에 전송
  • - 쿠폰의 변경이 있을 시 동기화를 위해 쿠폰이 적용된 딜번호를 message queue에 전송
  • - meta channel 쿠폰정보를 갱신하는 Batch를 통해 전달받은 queue를 소비 및 갱신

Project 02.

Targeting Coupon System

쿠폰은 사용자가 필요에 의해 버튼을 통해 발급받는게 일반적입니다.
타겟팅 쿠폰 시스템은 각자의 시스템을 통해 쿠폰을 꽂아주도록 설계된 시스템입니다.
즉, 다양한 쿠폰의 혜택을 사용자의 구매 패턴에 따라 구분하여 자동 발급합니다.
발급받지 않은 쿠폰이지만, 시스템을 통해 쿠폰을 발급함으로써 고객 구매를 유도할 수 있습니다.

운영 및 유지보수 기간 : 6개월

약 4천만장의 쿠폰을 일괄 발급하면 시스템의 물리적인 시간으로도 1~2일이 소요됬습니다.
쿠폰의 오발급, 누락, 실패 등을 방지하기 위해 트랜잭션 간 interval을 두었습니다.

  • - hive의 데이터는 Spring Batch를 통해 hiveQL을 사용하여 데이터를 가져옴
  • - 파티셔닝을 통해 데이터를 가져오며, mysql db에 각 파티션별 중간테이블을 생성
  • - 각 중간 테이블 정보를 기반으로 쿠폰발급을 발급하며 완료 후 중간테이블 삭제

Project 03.

PomPom SNS System

졸업한 K대학의 교내 SNS를 만들었고, 사이트 이름은 POMPOM으로 지었습니다.
교내에는 자신의 학과 외 타과 및 동아리, 교내소식 등을 교류할 수 있는 공간이 없었습니다.
그래서 교내 SNS를 만들고자 했으며, PC와 모바일웹 지원 및 페이스북 인증 가입을 통해
간편하고 빠른 접근성과 보안성을 동시에 높일 수 있었습니다.
오픈과 동시에 교내 600명 이상의 회원을 확보하며, 다양한 교류를 할 수 있도록 제공 했습니다.

개발기간 : 3 ~ 4개월

POMPOM

User Interface를 고려하여 PC와 모바일 웹으로 서비스를 구성하였습니다.
사용자들은 개개인의 타임라인, 친구들의 게시글이 스크롤방식으로 제공되는 뉴스피드
자신의 팔로잉을 관리 할 수 있는 친구 관리 및 관심사나 개인정보에 따른 친구추천 화면
그리고 메인 홈 페이지에서는 학과의 기숙사, 학생식당, 버스정보 등 다양한 교내 정보를
단순히 보는것뿐 아니라 유저들의 피드백을 통해 관리 될 수 있도록 구성했습니다.

POMPOM SNS는 기본적으로 사용자 수가 교내 수준이며, 빠르게 서버를 구현기위해
node js를 server side language로 선택했습니다. 또한 다양하게 제공되는 npm 모듈을 통해
여러가지 기능을 손쉽게 제공 받을 수 있었으며, 특히 express module을 중심으로 처리했습니다.
가입절차는 페이스북 연동과 교내 학번 연동을 통해, 간편한 프로세스로 가입이 가능했습니다.
이를 통해, 접근성과 편리성을 고려함과 동시에 개인정보에 대한 이슈도 해결할 수 있었습니다.
화면구성은 bootstrap을 기본으로하여, 필요한 css, js component를 import하여 사용했습니다.
교내 사이트에서 제공되는 식단, 버스정보 등을 crawling 하고 데이터를 가공함으로써
제공된 정보로 사용자들끼리 다양한 의견을 주고 받을 수 있는 공간을 만들어줬습니다.
