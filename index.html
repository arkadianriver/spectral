Project 01.

Coupon Admin System

또한 운영의 편의를 고려한 화면구성을 통해 업무효율을 증대 시킬 수 있었습니다.

소요기간 : 신규(1~2달) / 고도화(1~2달) / 보수(2주~1달)

Coupon Admin System

쿠폰 어드민은 운영의 편리성을 고려한 단순하고 명확하게 화면을 설계했으며
쿠폰에 적용되는 복잡하고 다양한 정책정보를 쉽게 컨트롤할 수 있게 구성했습니다.
쿠폰 생성/수정 시, 사용 용도에 따라 저장소를 다르게 사용했습니다.
각 저장소의 고유 특징을 통해, 쿠폰의 다양한 정책정보를 용도에 맞게 저장합니다.

  • mysql
  • - mysql은 내부 관리 데이터 저장소로, 가장 최신의 원천 데이터를 관리
  • - spring server에서 mybatis mapper를 통한 sql 관리

  • redis
  • - redis clients jedisCluster를 사용하여 레디스 연결
  • - 쿠폰의 수량, 적용대상, 만료 번호 등을 저장하여 각종 api, batch를 위한 데이터 저장
  • couchbase
  • - 쿠폰의 혜택, 계산된 가격 정보 등을 Json Document 기반으로 저장 및 관리
  • - user client side에서 사용 될 데이터 정보들을 저장
  • rabbit mq
  • - 쿠폰의 갱신시, meta channel(제휴사)에 노출되는 상품의 쿠폰에 대해 동기화가 필요
  • - 쿠폰의 변경이 있을 시 동기화를 위해 쿠폰이 적용된 딜번호를 message queue에 전송

다양한 쿠폰의 혜택을 니즈 패턴에 따라 구분하여 시스템에서 직접 발급합니다.

운영 및 유지보수 기간 : 6개월

Behind Coupon System

적게는 몇십만, 많게는 약 4천만장의 쿠폰을 일괄 발급 진행했습니다.
쿠폰에따라 전체 고객 대상으로 쿠폰을 발급할 경우도 종종 있습니다.
대용량 처리를 위해 hadoop driver hive 를 사용했습니다.


  • - hadoop driver hive를 통해 대량의 고객쿠폰 데이터를 적재
  • - 내부적으로 데이터를 파티셔닝 및 가공하여 hive에 중간테이블 데이터 적재
  • - batch를 통해 중간테이블 정보로 쿠폰발급을 진행, 완료 후 중간테이블 삭제
  • - 각각의 발급정보를 쿠폰발급 및 로그 테이블에 적재

Project 03.

PomPom SNS System

교내에는 자신의 학과 외 타과 및 동아리, 교내소식 등을 교류할 수 있는 공간이 없었습니다.
졸업한 K대학의 교내 SNS를 만들었고, 사이트 이름은 POMPOM 입니다.
오픈과 동시에 교내 600명 이상의 회원을 확보하며, 다양한 교류를 할 수 있도록 제공 했습니다.

소요기간 : 3 ~ 4개월


More

POMPOM

서비스는 PC와 모바일 웹으로 구성하였습니다.
사용자들은 개개인의 타임라인, 친구들의 게시글이 스크롤방식으로 제공되는 뉴스피드
자신의 팔로잉을 관리 할 수 있는 친구 관리 및 관심사나 개인정보에 따른 친구추천 화면
학과의 기숙사, 학생식당, 버스정보 등 다양한 교내 정보를 제공했습니다.
단순히 보는것뿐 아니라, 유저들의 소통 공간으로 구성했습니다.


  • - 사용자 수가 교내 수준이며, 빠르게 서버를 구현기위해 nodejs express로 개발
  • - sns 연동을 통한 회원관리 및 시스템 구성
  • - 교내 사이트에 제공되는 식단 버스정보 등 데이터를 크롤링하여 가공 및 제공
  • - 제공되는 모든 서비스에는 유저들이 소통가능한 시스템으로 구성

Roulette Promotion

앱전용 서비스로 진행하였으며, 모바일 웹으로 개발 후 모바일 웹뷰로 구성했습니다.

  • - 일2회, 주6회의 현 시스템 스케줄에 국한되지않는 범용 구조 설계
